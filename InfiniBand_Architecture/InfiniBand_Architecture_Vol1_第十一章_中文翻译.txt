第十一章：软件传输 Verbs
================================================================================

11.1 Verbs 介绍与概述

本章描述的 Verbs 提供了主机通道接口（host channel interface）向主机提供的功能的抽象定义。符合本规范的主机 CI 必须表现出 Verbs 所描述的语义行为。

由于 Verbs 定义了主机 CI 的行为，它们可能影响软件结构的设计，例如提供对主机 CI 访问的应用程序编程接口（API）。然而，本规范明确不定义任何此类 API。特别是，不要求与合规主机 CI 一起使用的 API 在语义上与 Verbs 保持一致。

11.1.1 Verb 扩展

本版本规范向 InfiniBand 架构规范 Volume 1 Release 1.1 添加了以下 Post-1.1 Verb 扩展列表：

• 基础队列管理扩展（Base Queue Management Extensions）：
  - 允许向发送队列或接收队列提交工作请求列表
  - 允许每个 HCA 有多个完成事件处理程序

• 共享接收队列（Shared Receive Queue）：
  - 允许多个 RC QP 或 UD QP 通过使用共享接收队列来减少接收资源数量

• 基础内存管理扩展（Base Memory Management Extensions）：
  - 允许消费者分配内存区域资源以供将来注册使用
  - 允许消费者通过发送队列执行非流水线的本地内存区域注册
  - 允许消费者通过发送队列执行非流水线的已注册内存区域和内存窗口的本地无效化
  - 允许消费者执行内存区域和内存窗口的远程无效化
  - 通过使用保留 L_Key 允许 HCA 直接访问物理地址
  - 对现有内存管理 verbs 提供以下补充：
    · 允许内存窗口与单个 QP 关联
    · 允许消费者拥有内存区域的 L_Key 字段的密钥部分以及内存区域和内存窗口的 R_Key 的密钥部分
    · 明确定义内存区域是共享的还是非共享的

注：对于队列操作，非流水线操作可以在发布到同一队列的前面的操作完成之前执行。

• 块列表物理缓冲区列表（Block List based Physical Buffer List），要求同时支持基础内存管理扩展。

• 零基虚拟地址（Zero Based Virtual Address, ZBVA），要求同时支持基础内存管理扩展。

• 本地无效化围栏（Local Invalidate Fencing），要求同时支持基础内存管理扩展。

o11-0.2.1: 如果 HCA 支持某个扩展，则它必须支持本规范中与该扩展相关的所有 verbs 和修饰符。

11.1.2 Verb 类别

11.1.2.1 强制性与可选 Verbs

某些 Verbs 是强制性的，某些仅在支持可选功能时才需要。

C11-1: CI 应支持 Verb 类别中分类为强制性的所有 Verbs。

C11-2: 如果 CI 声称符合某个可选功能，则 CI 应支持 Verb 类别中与该可选功能相关的所有 Verbs。

11.1.2.2 强制性与可选 Verb 功能

某些 Verbs 定义了仅在支持某些可选功能时才适用的功能。

C11-3: 如果 CI 支持给定的 Verb，则 CI 应支持为该 Verb 定义的所有未标示为可选的功能。

C11-4: 如果 CI 支持给定的 Verb 并声称符合某个可选功能，则 CI 应支持为该 Verb 定义的与该可选功能相关的所有功能。

11.1.2.3 消费者可访问性

Verb 消费者是 Verbs 的直接用户，分为两类：特权级（Privileged）和用户级（User-Level）。

特权级消费者通常是那些在足以直接访问 OS 内部数据结构的特权级别上运行的消费者，并且负责控制对通道接口的访问。所有 Verbs 都可供特权级消费者使用。

用户级消费者是那些必须依赖具有足够高特权级别的另一个代理来操作 OS 数据结构的消费者。只有那些明确标注为用户级的 Verbs 才可供用户级消费者使用。

11.2 传输资源管理

11.2.1 HCA

11.2.1.1 Open HCA（打开 HCA）

描述：
打开指定的 HCA 并返回一个不透明的对象或句柄，以唯一引用每个 HCA，使消费者能够区分终端节点中的各个 HCA。

C11-5: 系统中为不同 HCA 返回的句柄应全部唯一。

一旦打开，特定的 HCA 在关闭之前不能再次打开。打开 HCA 为消费者准备 HCA 供使用。

C11-6: 如果对当前已打开的 HCA 调用 Open HCA，CI 应返回"HCA 已在使用中"错误。

HCA 可以以块模式（block mode）或页面模式（page mode）打开，但不能同时以两种模式打开。所有 HCA 都必须支持页面模式。如果以块模式打开 HCA 但该 HCA 不支持块模式，Open HCA 必须返回即时错误。

输入修饰符：
• 此 HCA 的唯一标识符。命名方案由 OSV 定义。
• 将在 HCA 上使用的物理缓冲区类型：块（Block）或页面（Page）。

输出修饰符：
• 用于此 HCA 实例的句柄，用作其他 Verbs 的修饰符以指定所需的目标 HCA。

Verb 结果：
• 操作成功完成。
• 资源不足，无法完成请求。
• 无效的 HCA 名称。
• HCA 已在使用中。
• 不支持块类型物理缓冲区。

11.2.1.2 Query HCA（查询 HCA）

描述：
返回指定 HCA 的属性。

本节中定义的最大值是保证不超过的值。实现可能从相同的空间分配某些 HCA 资源。在这种情况下，返回的最大值不保证所有这些资源同时可用。

输入修饰符：
• HCA 句柄。

输出修饰符：
返回的 HCA 属性包括：

• 供应商特定信息，例如：
  - 供应商 ID（Vendor ID）。
  - 供应商提供的部件 ID（Part ID）。
  - 硬件版本。
  这三项供应商特定信息分别对应于组件 NodeInfo:VendorID、NodeInfo:DeviceID 和 NodeInfo:Revision 中的信息（参见第 14.2.5.3 节）。

HCA 特定值：
• 此 HCA 支持的最大 QP 数量。
• 此 HCA 支持的任何工作队列上的最大未完成工作请求数。
• 此 HCA 支持的每个工作请求的最大分散/聚集条目数（适用于可靠数据报接收队列工作请求以外的所有工作请求）。
• 此 HCA 支持的每个可靠数据报接收队列工作请求的最大分散/聚集条目数。如果不支持 RD 服务则为零。
• 此 HCA 支持的最大 CQ 数量。
• 此 HCA 支持的每个 CQ 的最大 CQE 容量。
• 此 HCA 支持的最大内存区域数量。
• 此 HCA 可注册的最大连续块（以字节为单位）。
• 此 HCA 支持的最大保护域数量。
• 此 HCA 支持的内存页大小。
• 此 HCA 支持地址句柄端口号检查的能力。
• 此 HCA 上的物理端口数量。

端口属性列表（此 HCA 上的每个端口一个列表）：
• 此 HCA 每个端口支持的 MTU 和消息大小。MTU 包含与该端口的 PortInfo:MTUCap 组件中找到的相同信息（参见第 14.2.5.6 节）。
• 每个端口的基本 LID 和 LMC。这些值仅在端口状态为 Armed 或 Active 时有效。对于其他端口状态，返回的值是不确定的。（有关端口状态的更多信息，参见 14.4.5 端口状态转换第 877 页）。
• 源 GID 表的内容和长度。已分配 GID 的值仅在端口状态为 Armed 或 Active 时有效。对于其他状态，已分配 GID 的值是不确定的。
• 此 HCA 每个端口的 PortState。参见 7.2.7 状态机术语第 169 页。
• 分区表的内容和长度。每个端口需要一个分区表。分区表的内容仅在端口状态为 Armed 或 Active 时有效。对于其他状态，分区表的内容取决于实现。
• 此 HCA 支持的最大虚拟通道数。此值表示与该端口的 PortInfo:VLCap 组件相同的信息（参见第 14.2.5.6 节）。
• 可选的 Bad P_Key 计数器，用于 HCA 支持的每个端口。
• HCA 支持的每个端口的 Q_Key 违规计数器。
• 可选的 InitTypeReply 值（参见 14.2.5.6 节中的 PortInfo:InitTypeReply 组件）。如果不支持 InitTypeReply，则应为零。
• 此 HCA 每个端口的子网管理器地址信息内容。这是一个表，条目按每个 HCA 端口排列，包含该端口的子网管理器 LID 和服务级别。如果子网管理器尚未设置此信息（端口状态为 Armed 或 Active），则应设置为宽容 LID（0xFFFF）。
• 此 HCA 端口的子网超时值（参见 14.2.5.6 PortInfo）。
• 此 HCA 每个端口的以下 CapabilityMask 位，如 PortInfo CapabilityMask 中定义：
  - IsSM
  - IsSMDisabled
  - IsSNMPTunnelingSupported
  - IsDeviceManagementSupported
  - IsVendorClassSupported
  - IsClientReregistrationSupported
• 此 HCA 支持的最大分区数量。支持的分区数量必须至少为一。
• 此 HCA 的节点 GUID。
• 可选的系统映像 GUID（参见第 14.2.5.3 节 NodeInfo）。
• RNR-NAK 生成支持 RC 服务的指示符。
• 本地 CA ACK 延迟。此值指定本地 CA 从接收消息到发送关联 ACK 或 NAK 之间的最大预期时间间隔。建议用于计算 CM REQ 消息中的"本地 ACK 超时"字段或 CM REP 消息中的"目标 ACK 延迟"字段。以微秒为单位的延迟值使用 4.096µs * 2^(本地 CA ACK 延迟) 计算。延迟值不是 CA 响应时间的保证上限，而是可用作超时的"最大预期值"。
• 关闭端口能力支持指示符。
• InitType 支持指示符。指示是否支持 InitTypeReply 和设置 InitType 值的能力。
• 端口活跃事件支持指示符。
• 系统映像 GUID 支持指示符。
• Bad P_Key 计数器支持指示符。
• Q_Key 违规计数器支持指示符。
• 每个 QP 的最大响应方资源：此 HCA 作为目标时每个 QP 可以未完成的最大 RDMA 读取和原子操作数。仅当此 HCA 支持原子操作时才适用于原子操作。
• 每个 EEC 的最大响应方资源：此 HCA 作为目标时每个 EEC 可以未完成的最大 RDMA 读取和原子操作数。仅当此 HCA 支持 RD 和原子操作时才适用于原子操作。对于本版本规范，此值为一。
• 每个 HCA 的最大响应方资源：此 HCA 作为目标时用于 RDMA 读取和原子操作的最大资源数。仅当此 HCA 支持原子操作时才适用于原子操作。
• 每个 QP 的最大发起方深度：此 HCA 发起 RDMA 读取和原子操作的每个 QP 最大深度。仅当此 HCA 支持原子操作时才适用于原子操作。
• 每个 EEC 的最大发起方深度：此 HCA 发起 RDMA 读取和原子操作的每个 EEC 最大深度。仅当此 HCA 支持 RD 和原子操作时才适用于原子操作。对于本版本规范，此值为一。
• 此 HCA 支持原子操作的能力以及在自身和其他系统组件（如处理器和其他 HCA）之间序列化原子操作的能力。本版本规范为此定义了三个级别的原子性：
  - 不支持原子操作。
  - 仅保证此 HCA 上 QP 之间的原子性。
  - 保证此 HCA 与任何其他组件（如 CPU 和其他 HCA）之间的原子性。
• 此 HCA 可支持的最大 EE 上下文数量。如果 HCA 不支持可靠数据报，则为零。
• 此 HCA 支持的最大 RDD 数量。支持的 RDD 数量必须至少为两。如果 HCA 不支持可靠数据报，则为零。
• 此 HCA 支持的最大内存窗口数量。
• 此 HCA 支持的最大 Raw IPv6 数据报 QP 数量。如果不支持 Raw IPv6 数据报，则为零。
• 此 HCA 支持的最大 Raw Ethertype 数据报 QP 数量。如果不支持 Raw Ethertype 数据报，则为零。
• 此 HCA 支持修改每个 QP 最大未完成工作请求数的能力。
• 此 HCA 支持的最大多播组数量。如果此 HCA 不支持 IBA 不可靠多播，则为零。
• 可附加到此 HCA 多播组的最大 QP 数量。如果此 HCA 不支持 IBA 不可靠多播，则为零。
• 此 HCA 支持的每个多播组的最大 QP 数量。如果此 HCA 不支持 IBA 不可靠多播，则为零。
• 此 HCA 支持原始数据包多播的能力。
• 此 HCA 支持自动路径迁移的能力。
• 此 HCA 支持的最大地址句柄数量。
• 此 HCA 在从 SQD 到 SQD 状态转换时更改 QP 或 EEC 主端口的能力。
• 此 HCA 支持 Modify Queue Pair 的当前 QP 状态修饰符的能力。
• 此 HCA 支持每个内存区域多种物理缓冲区大小的能力。仅当不支持基础内存管理扩展时返回此输出修饰符。
• 此 HCA 支持基础内存管理扩展的能力。如果 HCA 支持基础内存管理扩展：
  - 保留 L_Key 的值。
  - 调用 Allocate L_Key verb 时此 HCA 支持的最大物理缓冲区列表大小。
  - 此 HCA 支持的页大小列表。
  - 绑定 Type 2 内存窗口关联机制：
    · Type 2A - QP 号关联；或
    · Type 2B - QP 号和 PD 关联。
• 此 HCA 当前使用的物理缓冲区类型：页面或块。
• 此 HCA 支持每个内存区域多种页大小的能力。
• 此 HCA 支持块列表物理缓冲区列表的能力。
• 此 HCA 支持的块大小范围。
• 此 HCA 支持零基虚拟地址的能力。
• 此 HCA 支持本地无效化围栏的能力。
• 此 HCA 支持基础队列管理扩展的能力。如果 HCA 支持基础队列管理扩展：
  - 最大 CQ 事件处理程序数量。
• 此 HCA 支持共享接收队列的能力。如果 HCA 支持共享接收队列：
  - 最大 SRQ 数量。
  - 每个 SRQ 的最大 WR 数量。
  - 每个 SRQ WR 的最大分散/聚集条目数。
  - 修改每个 SRQ 最大 WR 数量的能力。

Verb 结果：
• 操作成功完成。
• 资源不足，无法完成此请求。
• 无效的 HCA 句柄。

11.2.1.3 Modify HCA Attributes（修改 HCA 属性）

描述：
修改 HCA 中的某些属性，由 HCA 上每个端口的单独列表指定。

如果要修改的任何 HCA 属性无效，则不应修改任何 HCA 属性。应返回即时错误，HCA 状态应保持不变。

输入修饰符：
• HCA 句柄。
• 可选的系统映像 GUID。
• 端口属性列表（此 HCA 上每个端口一个列表）：
  - 可选的关闭端口指示符。如果设置了关闭端口指示符，则端口将转换到 Down 状态（参见第 7.2.7 节中的 PortState）。
  - 可选的 InitType 值（参见 14.2.5.6 节中的 PortInfo:InitType 组件）。
  - Q_Key 违规计数器重置位。此位仅在 HCA 支持 Q_Key 违规计数器时适用。当设置此位时，Q_Key 违规计数器应被清除。
  - 以下 CapabilityMask 位，如 PortInfo CapabilityMask 中定义：
    · IsSM
    · IsSNMPTunnelingSupported
    · IsDeviceManagementSupported
    · IsVendorClassSupported

输出修饰符：（无）

Verb 结果：
• 操作成功完成。
• 无效的 HCA 句柄。
• 指定的计数器无效。

11.2.1.4 Close HCA（关闭 HCA）

描述：
关闭并重置指定的 HCA。此 Verb 仅负责释放通道接口分配的用于准备 HCA 供消费者使用的资源。所有其他资源不再与 CI 关联或连接，由消费者负责按需处理。

输入修饰符：
• HCA 句柄。

输出修饰符：（无）

Verb 结果：
• 操作成功完成。
• 无效的 HCA 句柄。

11.2.1.5 Allocate Protection Domain（分配保护域）

描述：
分配一个未使用的保护域对象。创建队列对、共享接收队列、地址句柄、注册内存和分配内存窗口时需要保护域对象。保护域对象提供队列对、共享接收队列、地址句柄、内存区域和内存窗口之间的关联。

在不可靠数据报队列对上的操作仅在队列对的保护域与工作请求中包含的地址句柄的保护域相同时才允许。

输入修饰符：
• HCA 句柄。

输出修饰符：
• 保护域对象。

Verb 结果：
• 操作成功完成。
• 资源不足，无法完成请求。
• 无效的 HCA 句柄。

11.2.1.6 Deallocate Protection Domain（释放保护域）

描述：

11.2.4 队列对

11.2.4.1 Create Queue Pair（创建队列对）

描述：
为指定的 HCA 创建一个 QP。

消费者必须指定一组初始 QP 属性。

C11-7: 如果任何所需的初始属性非法或缺失，则应返回错误且不应创建队列对。

成功后，返回新创建的 QP 的句柄和 QP 号。

o11-0.2.3: 如果 HCA 支持 SRQ，CI 必须允许 UD 和 RC QP 与 SRQ 关联。

输入修饰符：
• HCA 句柄。
• SRQ 句柄（如果 QP 将与 SRQ 关联）。
• 在 QP 创建时必须指定的 QP 属性：
  - 与发送队列关联的 CQ。
  - 与接收队列关联的 CQ。
  - 消费者期望提交到发送队列的最大未完成工作请求数。
  - 消费者期望提交到接收队列的最大未完成工作请求数。如果 QP 与 SRQ 关联，此值将被忽略。
  - 消费者将在提交到发送队列的工作请求中指定的最大分散/聚集元素数。
  - 消费者将在提交到接收队列的工作请求中指定的最大分散/聚集元素数。如果 QP 与 SRQ 关联，此值将被忽略。
  - 与此 QP 关联的可靠数据报域。仅适用于 RD QP。
  - 必须为此 QP 的发送队列指定信号类型。有效类型为：
    · 不可选择（Non-selectable）：提交到发送队列的所有工作请求始终生成完成条目。
    · 可选择（Selectable）：消费者必须在提交到发送队列的每个工作请求上指定是否为成功完成生成完成条目。
  - 消费者必须指定保护域。
  - 此 QP 请求的传输服务类型。有效的服务类型为：
    · 可靠连接（Reliable Connection）。
    · 可靠数据报（Reliable Datagram）。
    · 不可靠连接（Unreliable Connection）。
    · 不可靠数据报（Unreliable Datagram）。
  - 启用或禁用快速注册 PMR 和保留 L_Key 操作。

输出修饰符：
• 新创建的 QP 的句柄。
• QP 号。
• 发送队列上支持的实际未完成工作请求数。如果未返回错误，保证大于或等于请求的数量。（这可能需要消费者增加 CQ 的大小。）
• 接收队列上支持的实际未完成工作请求数。如果未返回错误，保证大于或等于请求的数量。（这可能需要消费者增加 CQ 的大小。）如果 SRQ 与 QP 关联，则不返回此值。
• 可在提交到发送队列的工作请求中指定的实际分散/聚集元素数。如果未返回错误，保证大于或等于请求的数量。
• 可在提交到接收队列的工作请求中指定的实际分散/聚集元素数。如果未返回错误，保证大于或等于请求的数量。如果 SRQ 与 QP 关联，则不返回此值。

Verb 结果：
• 操作成功完成。
• 资源不足，无法完成请求。
• 无效的 HCA 句柄。
• 无效的 CQ 句柄。
• 请求的最大工作请求数超过 HCA 能力。
• 请求的最大分散/聚集元素数超过 HCA 能力。
• 无效的保护域。
• 此 QP 的服务类型无效。
• 无效的可靠数据报域。
• 无效的 SRQ 句柄。
• HCA 不支持基础内存管理扩展。
• HCA 不支持 SRQ。

11.2.4.2 Modify Queue Pair（修改队列对）

描述：

C11-8: 调用此 Verb 时，CI 应修改指定 QP 的属性，然后使 QP 转换到指定的 QP 状态。

只有 QP 属性的子集可以在每个 QP 状态中修改。

（后续为 Modify QP 的详细输入/输出修饰符和 Verb 结果，以及 Query QP、Destroy QP 等内容，将在下一部分继续。）


11.2.4.2 修改队列对（续）

输入修饰符：
• HCA 句柄。
• QP 句柄。
• 要修改的 QP 属性及其新值。QP 创建后可修改的 QP 属性包括：
  • 下一个 QP 状态。如果指定当前状态，则仅修改 QP 属性。
  • 启用或禁用发送队列排空（Send Queue Drained）异步关联事件通知。此修饰符仅在选择的下一个 QP 状态为 SQD 时适用。
  • 主 P_Key 索引。不适用于 Raw 数据报或可靠数据报 QP。
  • 对传入数据报消息进行检查的 Q_Key，并可能用作传出 Q_Key（基于 WR Q_Key）。仅适用于 UD 和 RD QP。
  • 发送 QP 的 PSN。仅适用于 RC、UD 和 UC QP。
  • 如果 HCA 支持工作队列大小调整，消费者期望提交到发送队列的最大未完成工作请求数。
  • 如果此 HCA 支持工作队列大小调整，消费者期望提交到接收队列的最大未完成工作请求数。如果 QP 与 SRQ 关联，则忽略此值。

以下属性不适用于指定的 QP 为特殊 QP 的情况：SMI QP（QP0）、GSI QP（QP1）、Raw IPv6 和 Raw Ethertype。
  • 与此 QP 关联的主物理端口。不适用于 RD QP。适用于 RC QP 的 SQD 到 RTS 转换（如果 HCA 支持）。适用于所有 QP 的 Init 到 Init 转换。
  • 接收 QP 的 PSN。仅适用于 RC 和 UC QP。
  • 启用或禁用此 QP 上的传入 RDMA 读取。不适用于不可靠服务类型 QP。
  • 启用或禁用此 QP 上的传入 RDMA 写入。不适用于 UD 服务类型 QP。
  • 启用或禁用此 QP 上的传入原子操作。不适用于不可靠服务类型 QP。
  • 目标 QP 编号。仅适用于 RC 和 UC QP。
  • 发起者深度（Initiator Depth）：任何时刻未完成的 RDMA 读取和原子操作数量。仅适用于 RC QP。
  • 响应者资源（Responder Resources）：用于处理传入 RDMA 读取和原子操作的响应者资源数量。此值可向上舍入到支持的值，但不超过此 HCA 允许的 QP 最大值。仅适用于 RC QP。
  • 最小 RNR NAK 定时器字段值。当消息到达且目标为本地接收队列，而该接收队列没有未完成的接收工作请求时，CI 可能会向发起者响应 RNR NAK 数据包。此修饰符是在此类 RNR NAK 数据包的定时器字段中发送的最小值；它不影响因其他原因发送的 RNR NAK。如果指定的值不是第 330 页表 45"RNR NAK 定时器字段编码"中定义的 RNR NAK 定时器字段值之一，CI 应返回即时错误。仅适用于 RC 和 RD QP。
  • 当前 QP 状态。此修饰符仅在移动到 RTS 状态时有效。
  • 地址向量，仅适用于 RC 和 UC 传输，包含：
    • 服务级别。
    • 发送全局路由头标志。
    • 目标 LID。如果目标在同一子网中，LID = 最终目标；否则 LID = 路由器 LID。
    • 路径 MTU。
    • 最大静态速率。
    • 本地 ACK 超时（参见第 678 页第 12.7.33 节"目标 ACK 延迟"和第 12.7.34 节"本地 ACK 超时"）。仅适用于 RC QP。
    • 重试计数。仅适用于 RC QP。
    • RNR 重试计数。仅适用于 RC QP。
    • 源路径位。
    • 对于全局目标：
      • 流量类别。
      • 流标签。
      • 跳数限制。
      • 源 GID 索引。
      • 目标 GID。
  • 备用路径地址信息，仅适用于此 CI 支持自动路径迁移时的 RC 和 UC QP。注意：备用路径的路径 MTU 必须与主路径相同。具体包括：
    • 备用路径 P_Key 索引。备用路径 P_Key 索引应映射到与主 P_Key 索引相同的分区密钥；否则 APM 可能无法正常工作。
    • 备用路径主物理端口。
    • 备用路径地址向量，包含：
      • 服务级别。
      • 发送全局路由头标志。
      • 目标 LID。如果目标在同一子网中，LID = 最终目标；否则 LID = 路由器 LID。
      • 最大静态速率。
      • 本地 ACK 超时（参见第 678 页第 12.7.33 节"目标 ACK 延迟"和第 12.7.34 节"本地 ACK 超时"）。仅适用于 RC QP。
      • 源路径位。
      对于全局目标：
      • 流量类别。
      • 流标签。
      • 跳数限制。
      • 源 GID 索引。
      • 目标 GID。
  • 路径迁移状态。仅在此 HCA 支持自动路径迁移时有效。可设置的有效状态为：
    • 已迁移（Migrated）。
    • 重新配备（Rearm）。

输出修饰符：
• 如果 HCA 支持工作队列大小调整，发送队列上支持的实际未完成工作请求数量。如果未返回错误，此值保证大于或等于请求的数量。（这可能需要消费者增加 CQ 的大小。）
• 接收队列上支持的实际未完成工作请求数量。如果未返回错误，此值保证大于或等于请求的数量。（这可能需要消费者增加 CQ 的大小。）如果 SRQ 与 QP 关联，则不返回此值。

Verb 结果：
• 操作成功完成。
• 资源不足，无法完成请求。
• 无效的 HCA 句柄。
• 无效的 QP 句柄。
• 无法更改 QP 属性。
• 不支持原子操作。
• P_Key 索引超出范围。
• P_Key 索引指定了 P_Key 表中的无效条目。
• 无效的 QP 状态。
• 无效的路径迁移状态。
• 超出 HCA 端口的 MTUCap。
• 无效的端口。
• 此 QP 的服务类型无效。
• 请求的最大工作请求数超出 HCA 能力。
• 无效的 RNR NAK 定时器字段值。
• WQ 上的未完成条目多于指定的大小。
• QP 仍有 Type 2A MW 绑定在其上。

11.2.5 获取特殊 QP

描述：
返回指定 HCA 端口上指定 QP 类型的句柄。特殊 QP 类型包括：SMI QP（QP0）、GSI QP（QP1）、Raw IPv6 和 Raw Ethertype。

C11-13：此 Verb 必须支持 QP0 和 QP1。

HCA 对两种 Raw 数据报类型的支持是可选的。

o11-1：如果 HCA 支持 Raw 数据报 QP 类型，此 Verb 也必须支持它们。

C11-14：与 SMI QP 和 GSI QP 关联的句柄，对于每个 HCA 端口的每个 QP 只能发放一次。在没有中间的 Destroy QP 的情况下，后续调用此 Verb 必须返回错误。

单个 QP 每端口的限制不适用于任一 Raw 数据报 QP 类型。

o11-2：如果支持 Raw 数据报服务，每端口支持的 Raw 数据报类型 QP 数量应由 Query HCA Verb 返回。

在从此 Verb 返回之前，将设置指定 QP 类型的特定实现所需的任何固定 QP 属性。例如，可能需要为 QP 初始化适当的传输服务类型。

C11-15：SMI/GSI QP 不得与任何非 SMI/GSI QP 共享完成队列。尝试这样做将导致无效的 CQ 句柄错误。

输入修饰符：

• HCA 端口号。

  • SMI QP（QP0）。
  • GSI QP（QP1）。
  • Raw IPv6。
  • Raw Ethertype。
• 与发送队列关联的 CQ。
• 与接收队列关联的 CQ。
• 消费者期望提交到发送队列的最大未完成工作请求数。
• 消费者期望提交到接收队列的最大未完成工作请求数。
• 消费者期望在提交到发送队列的工作请求中指定的最大分散/聚集元素数。
• 消费者期望在提交到接收队列的工作请求中指定的最大分散/聚集元素数。
• 此 QP 上发送队列的信号类型。有效类型为：
  • 提交到发送队列的所有工作请求始终生成完成条目。
  • 消费者必须在提交到发送队列的每个工作请求上指定是否为成功完成生成完成条目。
• 保护域。

输出修饰符：

• 通过 Verbs 在接收队列上支持的实际未完成工作请求数。如果未返回错误，此值保证大于或等于请求的数量。（这可能需要消费者增加 CQ 的大小。）
• 提交到发送队列的工作请求中可指定的实际分散/聚集元素数。如果未返回错误，此值保证大于或等于请求的数量。
• 提交到接收队列的工作请求中可指定的实际分散/聚集元素数。如果未返回错误，此值保证大于或等于请求的数量。

Verb 结果：

• 资源不足，无法完成请求。

• 可用的 Raw 数据报 QP 数量已超出。
• 无效的端口。
• 无效的 CQ 句柄。
• 请求的最大工作请求数超出 HCA 能力。
• 请求的最大分散/聚集元素数超出 HCA 能力。

EE 上下文属性可以在使用 EE 句柄的工作请求未完成时修改，但如果修饰符被更改，任何此类工作请求可能无法正确执行。

当 EE 处于 Init、RTR 和 RTS 状态时，接收队列上可以有未完成的 WR，当 EE 处于 RTS 状态时，发送队列上可以有未完成的 WR。如果修改 EE 修饰符，QP 上的任何未完成工作请求可能无法按预期执行。

如果要修改的任何 EE 属性无效或请求的状态转换无效，则不应修改任何 EE 属性。应返回即时错误，EE 状态应保持不变。

11.2.8.1 分配 L_Key

描述：

• HCA 句柄。
• 保护域。
• 请求分配的 PBL 资源大小。
• 请求远程访问。请求返回 R_Key。

• R_Key - 用于远程访问。仅在请求远程访问时返回 R_Key。
• 分配的 PBL 资源的实际大小。分配的 PBL 资源的实际大小必须等于或大于请求的 PBL 资源大小。

Verb 结果：

11.2.8.2 注册内存区域

描述：

从此 verb 返回的 L_Key 和（如果请求的）R_Key 由 CI 拥有。

输入修饰符：
• HCA 句柄。

• 要注册区域的长度（以字节为单位）。
• 分配给已注册区域的保护域。
• 访问控制 - 可选择以下任意组合，除非另有说明：
  • 启用本地写入访问。

  • 启用远程读取访问。
  • 启用远程原子操作访问（如果支持原子操作）。远程原子操作访问要求本地写入访问。

• VA 类型：
  • 虚拟地址。
  • 零基虚拟地址。

• 内存区域句柄 - 用于向内存管理 Verbs 标识此特定注册区域。
• L_Key - 用于本地访问。
• R_Key - 用于远程访问。仅在请求远程访问时返回 R_Key。

描述：
从 HCA 转换表中移除一个内存区域。如果在关联的注册 Verb 中锁定了该区域，则解除锁定。此 Verb 仅负责释放作为关联注册操作一部分分配的资源。所有其他资源是消费者的责任。

消费者在内存区域仍有绑定的内存窗口时尝试注销内存区域是错误的。通道接口实现有关于如何处理此错误的选项，描述在第 502 页 10.6.7.2.6"注销具有绑定窗口的区域"中。

正在处理且正在主动引用被注销内存区域中内存位置的工作请求或远程操作请求必须因保护违规而失败。尝试访问已注销内存区域中内存位置的工作请求或远程操作请求必须因保护违规而失败。

此 Verb 依赖于 OSV 提供的函数来执行内存页的解锁（unpinning）。

输入修饰符：
• HCA 句柄。
• 内存区域句柄 - 在区域注册时发放。

输出修饰符：
无输出修饰符。

Verb 结果：
• 操作成功完成。
• 无效的 HCA 句柄。
• 无效的内存区域句柄。
• 操作被拒绝；区域仍有绑定的窗口。

11.2.8.6 重新注册内存区域

描述：
修改现有内存区域的属性。消费者拥有的任何现有区域都可以修改，无论最初由哪个 Verb 创建，或最近由哪个 Verb（如果有）重新注册。返回适合在工作请求中使用的内存区域描述，用于描述本地可访问的内存位置。当特别请求时，返回适合供入站 RDMA 和/或原子操作使用的内存区域描述。

此 Verb 在概念上执行注销内存区域后跟注册内存区域的功能。在可能的情况下，Verb 层以下的资源应被重用而不是释放和重新分配。此 Verb 可用于更改区域的访问权限和/或保护域，以及更改注册的内存位置。

此 Verb 的 L_Key 和 R_Key 输出修饰符必须用于替代先前为此区域发放的任何值。

此 Verb 依赖于 OSV 提供的函数来执行内存页的锁定和解锁以及创建代表内存区域的虚拟到物理转换。

消费者在内存区域仍有绑定的内存窗口时尝试重新注册内存区域是错误的。通道接口实现有关于如何处理此错误的选项，如第 502 页 10.6.7.2.6 所述。

C11-18：如果 CI 返回"操作被拒绝"（由于绑定的窗口）错误，CI 不得更改当前注册。
C11-19：如果 CI 返回"无效的 HCA 句柄"或"无效的内存区域句柄"错误，CI 不得更改当前注册（假设它确实存在）。
C11-20：如果 CI 返回任何其他错误，CI 应使"旧"和"新"注册都无效，并释放任何关联的资源。
C11-21：对于远程代理在重新注册过程中访问内存区域的错误情况，CI 必须呈现与注销操作后跟单独注册操作相同的语义。

从此 verb 返回的 L_Key 和（如果请求的）R_Key 由 CI 拥有。注意，作为输入修饰符传入的 L_Key 和任何伴随的 R_Key 可能由消费者拥有。但是，此 verb 成功完成后，CI 拥有 L_Key 和 R_Key（如果有请求）的密钥部分。

共享 MR 在此 verb 成功完成后变为非共享 MR（如果设置了更改转换输入修饰符）。否则共享 MR 保持为共享 MR。

输入修饰符：
• HCA 句柄。
• 内存区域句柄 - 在区域注册时发放。
• 更改请求类型 - 可选择以下任意组合，每个请求所需的输入修饰符列在下方：
  • 更改转换。
    所需输入修饰符：
    • 虚拟地址。
    • 长度。
    • VA 类型：
      • 虚拟地址。
      • 零基虚拟地址。
  • 更改保护域。
    所需输入修饰符：
    • 保护域。
  • 更改访问控制。
    所需输入修饰符：
    • 访问控制选项。

输出修饰符：
• 内存区域句柄 - 必须用于将来引用此内存区域。可能与之前的区域句柄相同，也可能不同。
• L_Key - 用于本地访问。
• R_Key - 用于远程访问。仅在请求远程访问时返回 R_Key。

Verb 结果：
• 操作成功完成。
• 资源不足，无法完成请求。
• 无效的 HCA 句柄。
• 无效的内存区域句柄。
• 无效的虚拟地址。
• 无效的长度。
• 无效的保护域。
• 无效的访问控制说明符。
• 操作被拒绝；区域仍有绑定的窗口。
• HCA 不支持 ZBVA。

使用示例：
a）要仅修改已注册区域的访问控制，应向 Verb 提供内存区域句柄、更改访问控制请求和新的访问控制选项输入修饰符。
b）要更改区域的地址转换，应向 Verb 提供内存区域句柄、更改转换请求和新的虚拟地址及长度输入修饰符。之前锁定的页面将被解锁，新的内存区域将被锁定和注册（如果请求的话还会绑定），使用区域的访问控制和保护域。之前的转换将根据需要被移除或替换。

11.2.8.7 重新注册物理内存区域

描述：
修改现有内存区域的属性。消费者拥有的任何现有区域都可以修改，无论最初由哪个 Verb 创建，或最近由哪个 Verb（如果有）重新注册。返回适合在工作请求中使用的内存区域描述，用于描述本地可访问的内存位置。当特别请求时，返回适合供入站 RDMA 和/或原子操作使用的内存区域描述。

此 Verb 在概念上执行注销内存区域后跟注册物理内存区域的功能。在可能的情况下，Verb 层以下的资源应被重用而不是释放和重新分配。此 Verb 可用于更改区域的访问权限和/或保护域，以及更改注册的内存位置。

注意，作为输入修饰符传入的 L_Key 和任何伴随的 R_Key 可能由 CI 拥有。如果消费者请求 L_Key 和 R_Key（如果有请求）的密钥部分的所有权，则在此 verb 成功完成后，消费者将拥有 L_Key 和 R_Key（如果有请求）的密钥部分。

此 Verb 的 L_Key 和 R_Key 输出修饰符必须用于替代先前为此区域发放的任何值。CI 可以使用以下三个输入修饰符中的任何一个来访问内存区域：内存区域句柄、L_Key 或 R_Key。不期望 CI 在这三个输入修饰符之间执行一致性检查。

消费者在内存区域仍有绑定的内存窗口时尝试重新注册内存区域是错误的。通道接口实现有关于如何处理此错误的选项，如第 502 页 10.6.7.2.6 所述。

C11-22：对于重新注册物理内存区域 Verb，CI 应符合第 599 页 11.2.8.6"重新注册内存区域"中包含的所有合规性声明。

共享 MR 在此 verb 成功完成后变为非共享 MR（如果设置了更改转换输入修饰符）。否则共享 MR 保持为共享 MR。

输入修饰符：
• HCA 句柄。
• 内存区域句柄。
• L_Key。
• R_Key。仅在 L_Key 有伴随的 R_Key 时提供。
• 更改请求类型 - 可选择以下任意组合，每个请求所需的输入修饰符列在下方：
  • 更改转换。所需输入修饰符：
    • 请求密钥所有权。消费者请求 L_Key 和 R_Key（如果有请求）的密钥部分的所有权。注意，如果消费者不请求密钥部分的所有权，则从此 verb 返回的 L_Key 和 R_Key（如果有请求）不能用于快速注册物理 MR。
    • 用于新 L_Key 和 R_Key 的密钥。
    • 物理缓冲区列表。
      每个物理缓冲区的起始物理地址。
      • 如果 PBL 是页面类型，每个缓冲区必须在 HCA 支持的页面边界处开始和结束。
      • 如果 PBL 是块类型，每个缓冲区可以从任意物理地址开始。
      页面大小。仅在 HCA 支持每个 MR 多种页面大小时使用。
    • 物理缓冲区大小。对于块类型物理缓冲区，列表中的所有块大小必须相同。如果 HCA 不支持每个 MR 多种页面大小，则列表中的所有页面大小必须相同。如果 HCA 支持每个 MR 多种页面大小，则不适用。缓冲区大小必须匹配 HCA 支持的缓冲区大小之一。
    • 列表中物理缓冲区的总数。
    • VA 类型：
      • 虚拟地址。
      • 零基虚拟地址。
    • 如果 VA 类型是 VA，则为消费者请求的区域第一个字节的 IOVA。注意，对于 ZBVA 不传递 IOVA。
    • 要注册区域的长度（以字节为单位）。
    • 区域起始 IOVA 在第一个物理缓冲区内的偏移量。
  • 更改保护域。
    • 分配给已注册区域的保护域。
  • 更改访问控制。
    访问控制 - 可选择以下任意组合，除非另有说明：
    • 启用本地写入访问。
    • 启用远程写入访问。远程写入访问要求启用本地写入访问。
    • 启用远程读取访问。
    • 启用远程原子操作访问（如果支持原子操作）。远程原子操作访问要求本地写入访问。
    • 启用内存窗口绑定。
    注意：始终隐含本地读取访问。

输出修饰符：
• 内存区域句柄 - 用于向内存管理 Verbs 标识此特定注册区域。
• I/O 虚拟地址 - 通道接口实际为区域第一个字节分配的 IOVA。对于支持基本内存管理扩展的 HCA，返回的 IOVA 与请求的 IOVA 相同。
• L_Key - 用于本地访问。
• R_Key - 用于远程访问。如果访问控制输入修饰符启用了任何远程访问，则返回 R_Key。
• 分配的 PBL 资源的实际大小。分配的 PBL 资源的实际大小应大于或等于消费者作为输入修饰符传入的 PBL 资源大小。

Verb 结果：
• 操作成功完成。
• 资源不足，无法完成请求。
• 无效的 HCA 句柄。
• 无效的物理缓冲区列表条目。
• 无效的长度。
• 无效的偏移量。
• 无效的保护域。
• 无效的访问控制说明符。
• HCA 不支持基本内存管理扩展。
• HCA 不支持 ZBVA。
• HCA 未以页面模式打开。
• HCA 未以块模式打开。
• HCA 不支持每个 MR 多种 PB 大小。

11.2.8.8 注册共享内存区域

描述：
给定一个现有的内存区域，创建一个与相同物理内存位置关联的新独立内存区域，其目的是使新内存区域尽可能共享 HCA 映射资源。通过对此 Verb 的重复调用，可以有任意数量的内存区域潜在地共享相同的 HCA 映射资源，所有这些都与相同的物理内存位置关联。此 verb 创建的内存区域与其他内存注册 verbs 创建的内存区域行为相同。

为新内存区域指定的虚拟地址、保护域和访问权限不需要与现有内存区域相同。长度按定义是相同的。

消费者提供一个请求的虚拟地址，与新内存区域的第一页关联，通道接口返回实际分配的虚拟地址。

从此 verb 返回的 L_Key 和（如果请求的）R_Key 由 CI 拥有。

输入修饰符：
• HCA 句柄。
• 内存区域句柄 - 已注册区域的句柄。
• 虚拟地址 - 消费者请求的缓冲区第一页的地址。
• 保护域。
• 访问控制选项。
• VA 类型：
  • 虚拟地址。
  • 零基虚拟地址。

输出修饰符：
• 内存区域句柄 - 新内存区域的句柄。
• 虚拟地址 - 通道接口实际为第一页分配的地址。
• L_Key - 用于本地访问。
• R_Key - 用于远程访问。在请求远程访问权限时返回 R_Key。

Verb 结果：
• 操作成功完成。
• 资源不足，无法完成请求。
• 无效的 HCA 句柄。
• 无效的内存区域句柄。
• 无效的保护域。
• 无效的访问控制说明符。
• HCA 不支持 ZBVA。

11.2.8.9 分配内存窗口

描述：
此 Verb 分配一个与保护域关联的内存窗口。分配时它不与任何内存区域固有关联。

输入修饰符：
• HCA 句柄。
• 分配给内存窗口的保护域。
• 内存窗口类型：Type 1 或 Type 2。

输出修饰符：
• 窗口句柄 - 用于向其他内存管理 Verbs 标识此特定内存窗口。
• R_Key - 未绑定的 R_Key，用于在绑定内存窗口 Verb 中指定窗口。

Verb 结果：
• 操作成功完成。
• 资源不足，无法完成请求。
• 无效的 HCA 句柄。
• 无效的保护域。
• HCA 不支持 Type 2 内存窗口。

11.2.8.10 查询内存窗口

描述：
此 Verb 返回与指定内存窗口关联的属性。

输入修饰符：
• HCA 句柄。
• 窗口句柄 - 由分配内存窗口发放。

输出修饰符：
• R_Key - 与内存窗口关联的当前 R_Key。
• 内存窗口类型：Type 1 或 Type 2。
• R_Key 状态。
• 与内存窗口关联的保护域。

Verb 结果：
• 操作成功完成。
• 无效的 HCA 句柄。
• 无效的内存窗口句柄。

11.2.8.11 绑定内存窗口

描述：
向指定的发送队列提交一个工作请求，该请求基于现有的内存区域将内存窗口绑定到指定的 VA 范围和远程访问属性。QP 服务类型必须是可靠连接、不可靠连接或可靠数据报。

指定的 VA 范围必须是整个内存区域或其子集。除非内存区域具有本地写入访问权限，否则不得指定远程写入访问或远程原子访问。QP、内存窗口和内存区域必须属于相同的 HCA 和保护域。

先前绑定的内存窗口可以绑定到相同或不同内存区域中的新 VA 范围，导致先前的绑定失效。将先前绑定的内存窗口绑定到零长度 VA 范围将使先前的绑定失效，并返回处于未绑定状态的 R_Key。

绑定操作有一个独特的排序规则：在绑定之后提交到发送队列的任何工作请求，在绑定操作完成之前不得开始执行。

在正常操作下，当远程代理正在访问内存窗口时，消费者更改内存窗口的绑定是不恰当的。但是，如果远程代理行为不当或在错误恢复情况下，这种情况可能会发生。当内存窗口的绑定更改时，正在处理且正在主动使用该内存窗口的任何远程操作请求必须因保护违规而失败。一旦绑定操作已报告给消费者已完成，通道接口必须保证在紧前一次绑定下不能执行额外的访问。

如果由于无效的输入修饰符无法完成绑定内存窗口操作，则先前的绑定内存窗口设置应保持有效，窗口的属性不应被修改。应返回即时错误或完成错误。

输入修饰符：
• HCA 句柄。
• QP 句柄。
• 工作请求，包含执行请求所需的信息。工作请求定义如下：
  • 用户定义的 64 位工作请求 ID。
  • 内存窗口句柄。
  • R_Key - 当前与内存窗口关联的 R_Key。
  • 内存区域句柄。
  • L_Key - 内存窗口将与之关联的内存区域的 L_Key。
  • 虚拟地址 - 绑定范围第一个字节的地址。虚拟地址的最大大小为 64 位。
  • 要绑定范围的长度（以字节为单位）。
  • 访问控制 - 可选择以下任意组合，除非另有说明：
    • 启用远程写入访问。要求内存区域具有本地写入访问。
    • 启用远程读取访问。
    • 启用远程原子操作访问（如果支持原子操作）。要求内存区域具有本地写入访问。
  • 完成通知指示器。如果发送队列以可选信号类型创建，则必须指定。如果发送队列以非可选信号类型创建，则忽略。
  • 栅栏指示器。如果设置了栅栏指示器，则队列上所有先前的 RDMA 读取和原子工作请求必须在开始处理此工作请求之前完成。栅栏指示器仅对可靠连接和可靠数据报传输服务有效。

输出修饰符：
• R_Key - 与新绑定关联的 R_Key，其值与提供的 R_Key 不同。

Verb 结果：
• 操作成功完成。
• 资源不足，无法完成请求。
• 无效的 HCA 句柄。
• 无效的 QP 句柄。
• 此 QP 的服务类型无效。
• 无效的内存窗口句柄。
• 无效的 R_Key。
• 无效的内存区域句柄。
• 无效的 L_Key。
• 无效的虚拟地址。
• 无效的长度。
• 无效的访问控制说明符。
• 提交的工作请求过多。
• 无效的 MW 类型。消费者尝试对 Type 2 内存窗口执行绑定操作。

11.2.8.12 释放内存窗口

描述：
在正常操作下，当远程代理正在访问内存窗口时，消费者释放内存窗口是不恰当的。但是，如果远程代理行为不当或在错误恢复情况下，这种情况可能会发生。当内存窗口被释放时，正在处理且正在主动使用该内存窗口的任何远程操作请求必须因保护违规而失败。释放 Verb 完成后，通道接口必须保证不能通过该内存窗口执行额外的访问。

输入修饰符：
• HCA 句柄。
• 窗口句柄 - 由分配内存窗口发放。

输出修饰符：
无输出修饰符。

Verb 结果：
• 操作成功完成。
• 无效的 HCA 句柄。
• 无效的内存窗口句柄。

11.3 多播

11.3.1 将 QP 附加到多播组

描述：
将 QP 附加到指定的多播组。此 Verb 的唯一功能是将此 QP 的接收工作队列分配给指定的多播组；附加完成后，此 QP 将获得每条寻址到由 MGID 指定的组并在与 QP 关联的 HCA 端口上接收的多播消息的副本。多播组的创建以及重新配置网络结构使寻址到该组的数据包被路由到本地 HCA 端口，描述在第 213 页 7.10"IBA 和原始数据包多播"中。

指定 QP 的服务类型必须是不可靠数据报。指定具有任何其他服务类型的 QP 是错误的。

允许将一个或多个 QP 附加到 HCA 上的多播组。如果当 QP 尝试附加到多播组时，HCA 的最大多播组附加数已达到上限，则返回错误。

如果尝试将特定 QP 附加到它已经附加的同一多播组，操作将明显成功（即返回操作成功完成）。但是，每条多播消息只会向附加的 QP 传送一份副本。

确定要附加到的多播组的输入修饰符是 MGID 和 MLID。两个输入修饰符都必须提供。

IBA 不可靠多播功能是可选的。仅当 HCA 支持 IBA 不可靠多播时才需要此 Verb。

输入修饰符：
• HCA 句柄。
• 多播组 MLID。
• 多播组 MGID。
• QP 句柄。

输出修饰符：

Verb 结果：
• 操作成功完成。
• 资源不足，无法完成请求。
• 无效的 HCA 句柄。
• 无效的多播 MLID。
• 无效的多播组 MGID。
• 无效的 QP 句柄。
• 此 QP 的服务类型无效。
• 附加到多播组的 QP 数量已超出。

描述：

输入修饰符：
• HCA 句柄。
• 多播组 MLID。
• 多播组 MGID。
• QP 句柄。

输出修饰符：

Verb 结果：
• 操作成功完成。
• 无效的 HCA 句柄。
• 无效的多播 MLID。
• 无效的多播组 MGID。

根据消费者提交的工作请求列表中包含的信息，为指定 QP 中的接收队列或 SRQ 构建一个或多个 WQE。这些 WQE 被添加到接收队列或 SRQ 的末尾，并通知 HCA 有一个或多个 WQE 准备好进行处理。

注意：消费者可以提交最多到 WQ 剩余容量的工作请求而不会遇到"提交的 WR 过多"即时错误。如果消费者提交超过剩余容量的请求，可能会返回即时错误。

C11-27.2.1：如果 CI 支持基本队列管理扩展，CI 应在工作请求列表被提交到接收队列后立即将控制返回给消费者。

如果返回即时错误，QP 状态或 SRQ 状态不受影响。

输入修饰符：
• HCA 句柄。
• QP 句柄。在向 QP 的接收队列提交且 QP 未与 SRQ 关联时使用。
• SRQ 句柄。在向 SRQ 提交时使用。
• 工作请求列表。每个条目由包含执行工作请求所需信息的工作请求组成。必须指定的 WR 修饰符取决于指定的操作类型。每个工作请求定义如下：
  • 用户定义的 64 位工作请求 ID。
  • 操作类型。接收队列唯一有效的操作是接收操作。
  • 分散/聚集列表。此列表可包含零个或多个数据段。

Verb 结果：

• 无效的 HCA 句柄。
• 无效的 QP 句柄。
• 提交的工作请求过多。
• 无效的操作类型。
• 无效的 QP 状态。
• 无效的分散/聚集列表格式。

• HCA 不支持基本队列管理扩展。
• 无效的 SRQ 句柄。
• QP 句柄用于与 SRQ 关联的 QP。

11.4.2.2 请求完成通知

描述：
请求在指定类型的下一个完成条目添加到指定 CQ 时调用 CQ 事件处理程序。每次请求完成通知调用对于特定 CQ 最多调用一次处理程序。在启用通知之前存在的任何 CQ 条目不会导致调用处理程序。

完成事件是两种类型之一：请求的或非请求的。当带有请求事件头位设置的传入 Send 或带即时数据的 RDMA Write 消息导致成功的接收工作完成被添加到 CQ 时，发生请求完成事件（Solicited Completion Event）；或者当不成功的工作完成被添加到 CQ 时也发生请求完成事件。当任何其他成功的接收工作完成或任何成功的发送工作完成被添加到 CQ 时，发生非请求完成事件（Unsolicited Completion Event）。

C11-30：CI 应支持请求和非请求完成事件类型。

当消费者请求完成通知时，必须指定是否对以下情况调用通知回调：
• 仅下一个请求完成事件，或
• 下一个请求或非请求完成事件。

C11-30.1.1：当"仅下一个请求完成事件"未决时，CI 应在以下任何条件发生时调用通知回调：
• 带有请求事件头位设置的传入 Send 导致成功的接收工作完成被添加到指定 CQ。
• 带有请求事件头位设置的传入带即时数据的 RDMA Write 导致成功的接收工作完成被添加到指定 CQ。
• 不成功的发送或接收工作完成被添加到指定 CQ。

C11-30.1.2：当"下一个请求或非请求完成事件"未决时，CI 应在任何工作完成被添加到指定 CQ 时调用通知回调。

如果请求完成通知正在等待中，对同一 CQ 的后续请求完成通知调用（在完成事件之前）仅影响通知何时发生。对下一个完成事件的请求完成通知优先于对同一 CQ 的请求完成事件的请求完成通知。

如果对同一 CQ 进行了多次请求完成通知调用，并且至少有一个请求将类型设置为下一个完成，则当下一个完成被添加到该 CQ 时将调用 CQ 事件处理程序。即使在指定 CQ 的完成事件之前发出了多个 CQ 通知请求，CQ 事件处理程序也只会被调用一次。

一旦 CQ 事件处理程序被调用，必须注册另一个完成通知请求，然后才会再次调用 CQ 事件处理程序。

C11-31：当 CQ 上存在对请求完成类型的完成通知请求，且对该 CQ 发出另一个指定下一个完成通知的请求时，CI 应将未决的完成通知类型更改为下一个完成。

C11-32：当 CQ 上存在对下一个完成的完成通知请求，且对该 CQ 发出另一个通知请求时，CI 不得更改未决的完成通知类型。

必须在调用此例程之前指定 CQ 事件处理程序。如果在生成事件时 CQ 事件处理程序尚未注册，则不会进行处理程序调用。

当 CQ 事件处理程序被调用时，它仅表示新条目已添加到指定的 CQ。HCA 和 CQ 句柄被传递给 CQ 事件处理程序，以便 CQ 事件处理程序可以确定是哪个 CQ 导致它被调用。

一旦处理程序例程被调用，消费者必须再次调用请求完成通知，才能在新条目添加到该 CQ 时得到通知。

消费者有责任调用轮询完成 Verb 来检索工作完成。

注意：如果消费者在没有与 CQ 关联的 CQ 事件处理程序 ID 的 CQ 句柄上请求完成通知，操作将无效。即不会生成完成事件。

输入修饰符：
• HCA 句柄。
• CQ 句柄。
• 请求的完成通知类型。类型为下一个完成或请求完成发生时。

输出修饰符：

Verb 结果：
• 操作成功完成。
• 无效的 HCA 句柄。
• 无效的 CQ 句柄。
• 无效的完成通知类型。

• 可靠数据报域正在使用中。
• 无效的可靠数据报域。
• 无效的 RNR NAK 定时器字段值。

内存操作错误：
• 无效的保护域。
• 保护域正在使用中。
• 无效的虚拟地址。
• 无效的长度。
• 无效的物理缓冲区列表条目。
• 无效的偏移量。
• 无效的 L_Key。
• 无效的 R_Key。
• 无效的内存区域句柄。
• 无效的内存窗口句柄。
• 无效的访问控制说明符。
• 操作被拒绝；区域仍有绑定的窗口。

多播错误：
• 无效的多播 MLID。
• 无效的多播组 MGID。

分区表错误：
• P_Key 索引超出范围。
• P_Key 索引指定了 P_Key 表中的无效条目。

Verb 扩展相关错误：
• HCA 不支持块类型物理缓冲区或未以此模式打开。
• HCA 未以页面模式打开。
• HCA 不支持基本内存管理扩展。
• HCA 不支持基本队列管理扩展。

• 无效的完成事件处理程序 ID。
• 无效的 MW 类型。
• 无效的 SRQ 句柄。
• SRQ 处于错误状态。
• SRQ 限制超出 SRQ 允许的最大工作请求数。
• SRQ 仍有与之关联的 QP。

11.6.2 完成返回状态

描述在工作请求处理过程中可能发生的工作完成状态错误返回结果。这些是在处理工作请求期间发生的错误，可以在工作完成状态中报告。

• 成功（Success）- 操作成功完成。
• 本地长度错误（Local Length Error）- 当提交到本地发送队列的工作请求中数据段长度之和超出通道适配器端口的消息长度时生成。当提交到本地接收队列的工作请求中数据段长度之和太小而无法接收有效的传入消息，或传入消息的长度大于接收消息的 HCA 端口支持的最大消息大小时生成。
• 本地 QP 操作错误（Local QP Operation Error）- 在处理此工作请求时检测到内部 QP 一致性错误。
• 本地 EE 上下文操作错误（Local EE Context Operation Error）- 在处理此工作请求时检测到内部 EE 上下文一致性错误。
• 本地保护错误（Local Protection Error）- 本地提交的工作请求的数据段未引用对请求操作有效的内存区域。
  • 关于保留 L_Key：
    • Verbs 消费者未启用保留 L_Key 访问。
• 工作请求刷新错误（Work Request Flushed Error）- 当 QP 转换到错误状态时，有工作请求正在处理或未完成。
• 内存管理操作错误（Memory Management Operation Error）- Verbs 消费者没有足够的权限来执行操作，原因包括：
  • 关于快速注册（Fast-Register）：
    • Verbs 消费者未启用快速注册访问；
    • 请求的 PBL 超出已分配 PBL 的大小；
    • 请求了远程访问，但 L_Key 没有伴随的 R_Key；或
    • 尝试对处于无效状态的 L_Key 或 R_Key 进行内存访问。
  • 关于内存窗口：
    • Verbs 消费者的访问权限不足；
    • 对 Type 2 内存窗口执行零长度绑定内存窗口或 Post Send Bind WR 操作；
    • Verb 消费者尝试将 MW 绑定到零基虚拟地址内存区域。
  • 关于无效化操作：
    • 尝试对处于无效状态的 L_Key 或 R_Key 进行内存访问；
    • 无法无效化内存区域，因为它是共享内存区域；
    • 无法无效化内存区域，因为它有绑定的内存窗口；或
    • 无法无效化内存区域，因为它是通过注册内存区域或重新注册内存区域创建的。
    • 无法无效化内存窗口，因为它是 Type 1 内存窗口。

以下错误仅对可靠 QP 报告。

• 错误响应错误（Bad Response Error）- 响应者返回了意外的传输层操作码。
• 本地访问错误（Local Access Error）- 在处理从远程节点发送的带即时数据的 RDMA Write 操作期间，在本地数据缓冲区上发生了保护错误。
• 远程无效请求错误（Remote Invalid Request Error）- 响应者检测到通道上的无效消息。可能的原因包括：此接收队列不支持该操作、缓冲不足以接收新的 RDMA 或原子操作请求，或 RDMA 请求中指定的长度大于 2^31 字节。
  在缓冲区大小不足以处理请求的情况下，传输到缓冲区的字节数是不确定的。但是，CI 不得写入超出缓冲区边界。
• 远程访问错误（Remote Access Error）- 在要被 RDMA 读取读取、被 RDMA 写入写入或被原子操作访问的远程数据缓冲区上发生了保护错误。此错误仅在 RDMA 操作或原子操作中报告。
• 远程操作错误（Remote Operation Error）- 响应者无法成功完成操作。可能的原因包括响应者 QP 相关错误阻止响应者完成请求，或接收队列上存在格式错误的 WQE。
• 传输重试计数器超出（Transport Retry Counter Exceeded）- 在尝试发送此消息时，本地传输超时重试计数器已超出。
• RNR 重试计数器超出（RNR Retry Counter Exceeded）- RNR NAK 重试计数已超出。

以下错误仅对 RD QP 或 EE 上下文报告。

• 本地 RDD 违规错误（Local RDD Violation Error）- 与 QP 关联的 RDD 与 EE 上下文关联的 RDD 不匹配。
• 远程无效 RD 请求（Remote Invalid RD Request）- 响应者检测到无效的传入 RD 消息。原因包括 Q_Key 或 RDD 违规。
• 中止错误（Aborted Error）- 操作被中止：
  • 对于 RD，请求者中止了操作。一种可能的原因是请求者暂停了操作并将稍后使用新的接收 WQE 重试。另一种可能的原因是请求者放弃了操作并将请求者 QP 置于 SQEr 状态。
  • 对于与 SRQ 关联的 UD QP，响应者中止了操作。
• 无效 EE 上下文编号（Invalid EE Context Number）- 检测到无效的 EE 上下文编号。
• 无效 EE 上下文状态（Invalid EE Context State）- 对于指定的 EE 上下文状态，操作不合法。

以上所有完成错误适用于通过 Post Send Request 和 Post Receive Request verbs 提交的 WR，内存窗口绑定错误状态除外。通过 Bind Memory Window verb 提交的 WR 必须以以下错误代码之一完成：成功、内存窗口绑定错误、本地 QP 操作错误或工作请求刷新错误。

11.6.3 异步事件

本节描述异步事件。异步事件分为四类：关联异步事件、关联异步错误、非关联异步事件和非关联异步错误。两种异步错误在第 531 页 10.10.2.3"异步错误"中定义。

关联异步事件分为两类，因为事件发生时 QP/EE 上下文的行为不同。

C11-34：当关联异步错误发生时，CI 应导致 QP/EE 转换到错误状态。

C11-35：当关联异步事件发生时，CI 必须让 QP/EE 保持在异步事件发生时所处的 QP/EE 状态。

非关联异步错误是那些无法与特定 QP 或 EE 上下文关联的错误。

Verbs 消费者必须按照设置异步事件处理程序中的描述注册处理程序，以便在异步事件发生时收到通知。此机制用于收集关于事件和错误的信息。

11.6.3.1 关联异步事件

关联异步事件是向 Verb 消费者发出的关于指定 QP 或 EE 上下文上发生了指定事件的建议。此类别中的事件不被通道接口视为错误，因此 QP/EE 状态保持不变。

• 路径已迁移（Path Migrated）- 表示连接已迁移到备用路径。
• 通信已建立（Communication Established）- 表示接收工作队列的第一个数据包已到达，而 QP/EE 仍处于 RTR 状态。作为此数据包目标的 QP/EE 的句柄在事件记录中返回。此事件可由通信管理器使用，如第 688 页 12.9.6"通信建立 - 被动端"中的状态图所示，并在第 12.9.7.2 节"被动状态"中描述。通信管理器可能在已处于已建立状态时接收此事件；这不是错误。

C11-36：对于 RC 和 UC 服务，如果第一个数据包在 QP 仍处于 RTR 状态时到达且第一个数据包被无错误地处理，CI 应生成通信已建立关联异步事件。允许 CI 在某些错误条件下生成此关联异步事件，但除非入站数据包验证至少在"actual PSN=ePSN?"框之后无错误，否则不得生成该事件（参见图 88、图 112）。

o11-5.1.1：如果 CI 支持 RD 服务，如果第一个数据包在 EEC 仍处于 RTR 状态时到达且第一个数据包被无错误地处理，CI 应生成通信已建立关联异步事件。允许 CI 在某些错误条件下生成此关联异步事件，但除非入站数据包验证至少在"actual PSN=ePSN?"框之后无错误，否则不得生成该事件（参见图 89）。

对于 UD 和 Raw 服务类型，允许但强烈不建议生成通信已建立关联异步事件。

• 发送队列已排空（Send Queue Drained）- 表示指定队列对或 EE 的发送队列已完成请求状态更改时正在进行的未完成消息，并且（如果适用）已收到这些消息的所有确认；如果到 SQD 的转换被到 SQError、Error 或 Reset 状态的转换中止，也会生成该事件。

o11-5.2.4：如果 HCA 支持 SRQ，对于 RC 和 UD 服务，如果 SRQ 限制事件生成机制已配备且达到了 SRQ 限制，CI 应生成 SRQ 限制已达到关联异步事件。每当 SRQ WQE 数量小于 SRQ 限制时，就达到了 SRQ 限制。
CI 应在事件生成后将 SRQ 限制重置为零。

o11-5.2.5：如果 HCA 支持 SRQ，对于 RC 和 UD 服务，当处于错误状态且与 SRQ 关联的 QP 出现以下情况时，CI 应生成最后 WQE 已达到关联异步事件：
• 为最后一个 WQE 生成了 CQE，或
• QP 进入错误状态且 RQ 上没有更多 WQE。

o11-5.2.6：如果 HCA 支持 SRQ，对于 RC 和 UD 服务，CI 在同一 QP 上出现本地工作队列灾难性错误后，不得在该 QP 上生成最后 WQE 已达到关联异步事件。
如果 HCA 遇到本地工作队列灾难性错误且最后 WQE 已达到关联异步事件未发生，则消费者必须销毁所有 QP 和 SRQ 以回收与该 QP 关联的所有 WQE。

11.6.3.2 关联异步错误

• CQ 错误（CQ Error）- 表示在向完成队列写入条目时发生错误。

C11-37：当在向 CQ 写入条目时发生除 CQ 溢出以外的错误时，CI 应生成 CQ 错误。

C11-38：当检测到 CQ 溢出时，CI 应生成 CQ 错误。

此条件将导致任何关联的工作队列在尝试使用该 CQ 时出现关联异步错误。不能再向 CQ 添加完成。不保证在错误发生时 CQ 中存在的完成可以被检索。可能的原因包括 CQ 溢出或 CQ 保护错误。

• 本地工作队列灾难性错误（Local Work Queue Catastrophic Error）- 在访问或处理工作队列时发生了阻止完成报告的错误。

C11-39：当与导致 CQ 错误的 CQ 关联的工作队列尝试使用该 CQ 时，CI 应生成本地工作队列灾难性错误。

C11-40：当在访问或处理工作队列时发生了阻止完成报告的错误时，CI 应生成本地工作队列灾难性错误。

• 无效请求本地工作队列错误（Invalid Request Local Work Queue Error）- 传输层在响应端检测到传输操作码违规。可能的原因为：不支持或保留的操作码；或操作码顺序错误。

C11-40.1.1：对于 RC 服务，当传输层在响应端检测到传输操作码违规时，CI 应生成无效请求本地工作队列错误。响应端的关联 QP 应被置于错误状态。

• 本地访问违规工作队列错误（Local Access Violation Work Queue Error）- 传输层在响应端检测到请求访问违规。可能的原因为：原子操作未对齐；RDMA Read 或原子请求过多；R_Key 违规；或无即时数据的长度错误。

C11-40.1.2：对于 RC 服务，当传输层在响应端检测到请求访问违规时，CI 应生成本地访问违规工作队列错误。响应端的关联 QP 应被置于错误状态。

• 本地 EE 上下文灾难性错误（Local EE Context Catastrophic Error）- 在访问或处理 EE 上下文时发生了阻止完成报告的错误。

o11-5.a1：如果 CI 支持 RD 服务，当在访问或处理 EE 上下文时发生了阻止完成报告的错误时，CI 应生成 EE 上下文灾难性错误。

• 路径迁移请求错误（Path Migration Request Error）- 表示此 QP/EE 的传入路径迁移请求未被接受。验证过程在"迁移请求"一节中定义。

o11-6：如果 CI 支持自动路径迁移，当此 QP/EE 的传入路径迁移请求未被接受时，CI 应生成路径迁移请求错误。

• SRQ 灾难性错误（SRQ Catastrophic Error）- 在处理或访问 SRQ 时发生了阻止从 SRQ 出列 WQE 和报告接收完成的错误。

o11-6.2.1：如果 HCA 支持 SRQ，当除"资源未就绪"以外的错误阻止 HCA 从 SRQ 出列 WQE 时，CI 应生成 SRQ 灾难性错误。当此错误发生时，CI 应将 SRQ 置于错误状态。

o11-6.2.2：如果 HCA 支持 SRQ，对于与 SRQ 关联的 QP，在以下任一情况下，CI 应生成本地工作队列灾难性错误并将 QP 置于错误状态：
• 除"资源未就绪"以外的错误阻止 HCA 从 SRQ 出列 WQE；或
• QP 尝试从 SRQ 出列 WQE，但关联的 SRQ 已处于错误状态。

11.6.3.3 非关联异步事件

• 端口激活（Port Active）- 当链路变为激活状态时发出。

o11-6.1.1：如果支持端口激活事件，当链路被声明为激活时，CI 应生成端口激活事件。

使用下面端口错误描述中的"可用"和"不可用"状态定义，"端口激活"事件在与 HCA 端口关联的链路从不可用状态转换到可用状态时生成。

• 客户端重新注册事件（Client Reregistration Event）- 当 SM 请求客户端重新注册时发出（参见第 881 页 14.4.11"客户端重新注册"）。

o11-6.2.3：如果 CI 表明端口支持客户端重新注册，当 SMA 从 SM 收到此请求时，CI 应生成客户端重新注册事件。

11.6.3.4 非关联异步错误

• 本地灾难性错误（Local Catastrophic Error）- 发生了无法归因于任何资源的错误，CI 行为不确定。

C11-41：当发生无法归因于任何资源的错误且 CI 行为不确定时，CI 应生成本地灾难性错误。

• 端口错误（Port Error）- 当链路被声明为不可用时发出。

C11-42：当链路被声明为不可用时，CI 应生成端口错误。端口错误不得影响 QP/EE 状态。

使用链路状态的定义，"不可用"状态被认为是：Down、Initialize 和 Armed。"可用"状态是 Active 和 ActDefer。当与 HCA 端口关联的链路从可用状态转换到不可用状态时，生成"端口错误"非关联异步错误。
